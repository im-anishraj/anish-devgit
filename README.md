### 🧠 &nbsp; 1 — Hybrid Search RAG Engine

<table>
<tr>
<td>

**Stack:** `Python` `LangChain` `OpenAI GPT-4` `FAISS` `BM25` `FastAPI` `Docker`

A production-grade Retrieval-Augmented Generation engine designed to query 500+ pages of technical documentation with measurably better accuracy than standard keyword search.

The core innovation is a **Reciprocal Rank Fusion (RRF)** layer that merges dense semantic search (FAISS) with sparse keyword search (BM25) — combining the strengths of both and eliminating their individual weaknesses.

| What | How |
|------|-----|
| Retrieval accuracy | **+40%** over keyword-only baseline |
| Chunking strategy | SemanticChunker — preserves context, no fixed-size splits |
| Hallucination control | GPT-4 instructed to cite source + page, or say "I don't know" |
| Deployment | Dockerized FastAPI — single `docker compose up` |

</td>
</tr>
</table>

### ⚙️ &nbsp; 2 — End-to-End MLOps Pipeline

<table>
<tr>
<td>

**Stack:** `Python` `XGBoost` `Scikit-learn` `MLflow` `SMOTE` `FastAPI` `Docker` `AWS EC2`

A complete MLOps pipeline that takes raw customer data and produces a deployed, versioned, monitored prediction API — the kind of system actual data teams use, not a Jupyter notebook that lives on a laptop.

| What | How |
|------|-----|
| Model accuracy | **92%** on held-out test set |
| Class imbalance | 1:10 ratio — corrected with SMOTE **on training data only** |
| Experiment tracking | MLflow — all params, metrics, and artifacts versioned |
| Deployment | FastAPI prediction endpoint on AWS EC2 via Docker |
| Explainability | SHAP feature importance — Tenure & Charges identified as top drivers |

The SMOTE-on-training-only detail matters: most tutorials get this wrong, leaking test distribution into training. This pipeline does not.

</td>

### 📈 &nbsp; 3 — Algorithmic Trading Monitor

<table>
<tr>
<td>

**Stack:** `React` `Node.js` `Express` `MongoDB` `WebSockets` `Chart.js` `Tailwind CSS`

A high-frequency financial analytics dashboard that streams live market tick data via WebSockets and auto-calculates technical indicators to surface buy/sell signals in real time — no page refresh required.

| What | How |
|------|-----|
| Data latency | **< 100ms** — raw `ws` library, not Socket.io |
